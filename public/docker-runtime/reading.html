CHAPTER 02 / BUILD IS NOT RUN

构建模板和启动实例，是两件事

这是最重要的分界线。看到 Built，只说明模板制作完成；看到 Started 或 Running，才说明进程已经启动。

MAKES AN IMAGE

docker build

读取 Dockerfile 与 build context，生成不可变的镜像层。

  • 可能拉取基础 Image
  • 执行 COPY、RUN 等指令
  • 复用没有变化的缓存层
  • 输出带标签的 Image
$ docker build -t demo:1.0 .
Successfully tagged demo:1.0
STARTS AN INSTANCE

docker run

基于已有 Image 创建 Container，然后启动它的主进程。

  • 增加 Container 可写层
  • 准备网络和挂载目录
  • 应用 namespaces 与 cgroups
  • 执行 ENTRYPOINT + CMD
$ docker run --name demo demo:1.0
PID 1: java -jar app.jar
CHAPTER 03 / A CONTAINER IS A PROCESS

容器不是一台完整的小电脑

它更接近一个被隔离起来的 Linux 进程。多个 Container 可以共享一个内核，但彼此看到不同的进程、文件和网络。

Container AJava · 独立 PID / FS / Net
Container BNginx · 独立 PID / FS / Net
共享 Linux Kernelnamespaces · cgroups · networking
Docker Desktop Linux VMmacOS 上的 Linux 容器实际运行在这里

主进程决定 Container 的生命

启动命令生成的进程会成为容器里的 PID 1。它运行，Container 就是 Running；它退出，Container 通常就变成 Exited。

为什么启动快？
因为每个 Container 不需要再启动一套完整 Guest OS。

CHAPTER 05 / COMPOSE

一个命令，描述并启动整套服务

Compose 读取 compose.yml，构建需要本地制作的 Images，复用 PostgreSQL、MinIO 等现成 Images，再创建并启动各个 Container。
